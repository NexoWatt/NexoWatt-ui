# Roadmap ab NexoWatt UI 0.8.48

## 0.8.48 – Mesh/Microgrid Leistungsgrenzen je Knoten

- Maximalleistung je Knoten verbindlich in CommandGuard auswerten.
- Min-/Max-Leistung je Bridge-Ziel auswerten.
- Prioritätslogik mit Leistungsgrenzen kombinieren.
- Betreiberansicht: Limitgrund je blockiertem Command anzeigen.

## 0.8.49 – Community-/Nachbarschaftsledger

- Lokale kWh-Zuordnung zwischen Mesh-Knoten vorbereiten.
- Remote-Node-Anteile im Local kWh Ledger referenzieren.
- Keine eichrechtsverbindliche Abrechnung, nur Betreiber-/Optimierungsbasis.

## 0.8.50 – Feldtest-Härtung

- Zwei-Instanzen-Test mit realen Ziel-Bridges dokumentieren.
- Fehlerreaktion und Diagnose für Offline-Peers verbessern.
- Installationsanleitung für separates Tailscale-Mesh finalisieren.
